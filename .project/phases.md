# Phase Plan

Goal: Build "Skill Smith" — an autonomous skill-authoring tool that turns a one-line project description (e.g. "AEM project with React", "Spring Boot REST + SOAP service with SQL") into a complete, tested set of Claude Agent Skills (SKILL.md folders) that make a coding agent an expert in that exact stack, with zero manual skill-hunting.

Mode: autonomous

- [x] 1. Foundation + Stage 0 intake — Scaffold the Node+Express+TS backend, React frontend, config file, README and workspace/ layout; build the claude -p invocation wrapper (stream-json parsing, retries, cost/usage capture) proven by a live 'say hi' round-trip; and ship Stage 0 intake+clarifier end-to-end (scoping call -> questions as chips -> answered scope persisted to scope.json) with Stages 1-5 shown pending.
- [x] 2. Stage 1 research — Decompose the answered scope into per-domain research subtasks, run them as parallel claude -p agents with WebSearch/WebFetch enabled, stream live per-domain status, and persist a structured versioned brief (key_apis/idioms/gotchas/version_notes/sources) to workspace/<job>/research/<domain>.json.
- [x] 3. Stage 2-3 design + generation — Add the skill-design call that proposes a skill-set plan (one-domain-per-skill, split-by-variant, with an approve/edit gate), then generation that writes each skill directory (SKILL.md + references/*.md + scripts/) into workspace/<job>/skills/<name>/ obeying the ground-truth rules.
- [>] 4. Stage 4 self-test loop — Add the differentiator: trigger-reliability testing (generate prompts, measure trigger rate, rewrite description until it clears the threshold) and capability grading (run a representative task with the skill loaded, grade against brief-derived assertions), saving report.json per skill and looping back to generation on failure with a capped iteration count.
- [ ] 5. Stage 5 package + results — Validate frontmatter/body-length/structure, package each skill (skill-creator package_skill.py if present, else zip to .skill), and build the results screen showing per-skill trigger rate, capability score, SKILL.md preview, sources, downloads, 'Download all', and install hints.
- [ ] 6. Polish + history + fixtures — Add the re-runnable job history, refine the live cost/call meter and graceful-failure/resume across all stages, write the full README and inline-documented config, and commit an example completed job as a fixture.

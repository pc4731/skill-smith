# Skill Smith — Project Brief

## Overview
Skill Smith turns a one-line project description (e.g. "Spring Boot REST + SOAP + SQL") into a tested, packaged set of **Claude Agent Skills**, using the **Claude Code CLI (`claude -p`, headless) as its engine**. Thin orchestration + live web app over that engine. **COMPLETE — all 6 phases done; the full pipeline runs end-to-end to a downloadable deliverable.**

## Roadmap (.project/phases.md) — all [x]
1 Foundation + Stage 0 intake · 2 Stage 1 research · 3 Stage 2 design + Stage 3 generation · 4 Stage 4 self-test · 5 Stage 5 package + results · 6 Polish: history + re-run + restart-reconciliation + meter + example fixture. (Phases auto-advanced scope→research→design[park for approval]→generate→self-test→package.)

## Architecture
npm-workspaces monorepo. backend (Express+TS) spawns `claude -p` via child_process, parses stream-json, persists all state under workspace/<jobId>/, streams over SSE. frontend (React+Vite+TS) seeds from GET /api/jobs/:id then subscribes to SSE; holds no unrecoverable state. Pipeline = 6 stages (0–5 all live). runStageN(ctx,jobId) background-runtime pattern is the seam: each stage reads prior on-disk artifacts, does its work, persists, and auto-advances to the next (design parks for approval). Stage 5 is deterministic (no claude calls). On boot, reconcileOrphans() flips jobs left mid-stage to failed.

## Tech stack
Node ≥20 (dev 22), TS strict. Backend Express 4 + zod 3 + archiver 7 (zip), run via tsx; vitest 2 + supertest. Frontend React 18 + Vite 5 + react-router-dom 6; vitest + RTL (jsdom) + @testing-library/user-event. Engine = external `claude` CLI (or ANTHROPIC_API_KEY + --bare). Default model claude-opus-4-8. No DB — disk JSON. SSE.

## Key decisions
Claude CLI engine (stateless; stages read prior artifacts from disk). Disk = source of truth (atomic job.json + replayable events.ndjson) → refresh/restart-safe. Cost guardrails: per-job ceiling 150 + parallelism semaphore + per-day ceiling + max description length + skill cap; Stage 5 deterministic so consumes no budget. Security: web ONLY in research, NO Bash in any claude-invoking stage; all paths slug-confined; generated scripts/ inert (never executed) + Stage-5 scanSkill rejects secrets/`curl|sh` before packaging; localhost bind; args-array spawn (no injection, incl. rerun's cloned description); secrets never logged/streamed/in-prompts. Stage-4 trigger JUDGE sees only name+description; EVAL_EXPECT gated behind selfTest.evalLabel (prod false). 'Resume' is honest restart-reconciliation to failed (no mid-stage resume). rerun never mutates the source job. Tests mock the CLI via SKILL_SMITH_CLAUDE_BIN → fake-claude.mjs.

## Data model (disk, per job)
workspace/<jobId>/: job.json {id,kind,status(active|awaiting_input|done|failed),description,createdAt,updatedAt,stages[],scope?,questions?,answers?,research?,design?,generation?,selftest?,results?,meter,note?}; events.ndjson; scope.json; research/<slug>.json; plan.json; skills/<slug>/ (SKILL.md + references/ + scripts/) + report.json; skills/<slug>.skill; all-skills.zip; results.json; raw/<callId>.ndjson. JobSummary (history) = {id,kind,description,status,createdAt,updatedAt,skillCount,cost,calls}.

## Build / run / test
npm install → npm run dev (backend 127.0.0.1:4000 + frontend :5173). Flow: describe → answer Stage 0 (or Use defaults) → research → approve plan → generation → self-test → packaging → Results (download .skill / Download all + install hints); History lists past jobs + Re-run. npm test (mocks CLI; backend 64 + frontend 30 green), npm run typecheck (tsc), npm run build. Docker compose → :8080. Stages 4/5/6 runtime-verified live (probes in .verify/).

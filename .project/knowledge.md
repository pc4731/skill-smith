# Skill Smith — Project Brief

## Overview
Skill Smith turns a one-line project description into a tested, packaged set of **Claude Agent Skills**, using the **Claude Code CLI (`claude -p`, headless) as its engine**. Thin orchestration + live web app over that engine. Built **phased**; **Phases 1–5 complete** (the full pipeline now runs end-to-end to a downloadable deliverable).

## Phased roadmap (.project/phases.md)
- [x] 1 — Foundation + Stage 0 intake/clarifier.
- [x] 2 — Stage 1 research: parallel per-domain `claude -p` (WebSearch/WebFetch) → versioned cited briefs research/<slug>.json.
- [x] 3 — Stage 2 design (plan + approve gate → plan.json) + Stage 3 generation (skills/<slug>/ + deterministic validateSkill).
- [x] 4 — Stage 4 self-test (trigger optimizer + capability grading + iterate-on-failure → skills/<slug>/report.json).
- [x] 5 — Stage 5 package + results (validate + safety-scan + zip each skill to .skill, results.json, Results screen + downloads).
- [ ] 6 — polish/history/fixtures.
Mode autonomous; phases auto-advance scope→research→design[park for approval]→generate→self-test→package. If a phase STOPs without advancing, mark it [x] in phases.md before `orchestrate_phases build=true`.

## Architecture
npm-workspaces monorepo. backend (Express+TS) spawns `claude -p` via child_process, parses stream-json, persists all state under workspace/<jobId>/, streams over SSE. frontend (React+Vite+TS) seeds from GET /api/jobs/:id then subscribes to SSE; holds no unrecoverable state. Pipeline = 6 stages; **Stages 0–5 implemented** (Stage 5 is the last functional stage — job completes status:done). runStageN(ctx,jobId) background-runtime pattern is the seam; each stage reads prior on-disk artifacts and auto-advances to the next.

## Tech stack
Node ≥20 (dev 22), TS strict. Backend Express 4 + zod 3 + **archiver 7** (zip packaging), run via tsx; vitest 2 + supertest. Frontend React 18 + Vite 5 + react-router-dom 6; vitest + RTL (jsdom). Engine = external `claude` CLI (or ANTHROPIC_API_KEY + bare). Default model claude-opus-4-8. No DB — disk JSON. SSE.

## Key decisions
Claude CLI engine (stateless; stages read prior artifacts from disk). Disk = source of truth (atomic job.json + replayable events.ndjson) → refresh/restart-safe. Cost guardrails (per-job ceiling 150 + semaphore + per-day ceiling + max description length + skill cap); **Stage 5 is deterministic so it consumes no budget**. Security: per-stage tools — web ONLY in research, NO Bash anywhere; all artifact paths slug-confined; generated scripts/ inert (never executed by Skill Smith); secrets never in prompts/SSE; localhost bind; args-array spawn. Stage 4 trigger JUDGE sees only name+description; EVAL_EXPECT answer label gated behind selfTest.evalLabel (prod false; test true). Capability check loads skill by INLINING SKILL.md (no headless --skill flag). **Stage 5 safety gate (scanSkill): before zipping, reject any skill file containing a hardcoded-secret pattern (AWS/OpenAI keys, PEM, key=literal) or shell-exfiltration/obfuscation (curl|wget ... | sh, base64 -d | sh, eval(atob() — a tripped skill is failed, NOT shipped (delivery boundary).** Packages exclude .cap/report.json; entry names controlled (no zip-slip); raw/ lives at job level so never packaged. Partial fail → done_with_warnings. Tests mock the CLI via SKILL_SMITH_CLAUDE_BIN → fake-claude.mjs (all stage modes + FAIL_DOMAIN/FAIL_SKILL/lowtrig/lowcap hooks).
